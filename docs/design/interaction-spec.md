# Alike 交互规范文档（Interaction Spec）

> **项目：** Alike — 汇聚天下牛马，总有人懂你的辛苦
>
> **角色产出：** `claude-designer`（UX/交互设计师）
>
> **定位：** 本文档定义**交互行为、状态、流程与可达性**，是 `component-spec.md`（组件结构/Props）与 `design-system.md`（视觉 Token）之间的桥梁。视觉落地色值/图标交由 `claude-visual`，本文只给**语义与建议**。
>
> **主题：** 暗色 Aurora 极光风为默认，支持切换亮色（`.light`）。
>
> **术语约定：**
> - **触发（Trigger）** — 用户什么操作触发交互
> - **反馈（Feedback）** — 系统给出的视觉/动效/文案响应
> - **可达性（A11y）** — 键盘可达、焦点管理、ARIA、屏幕阅读器
> - 状态统一枚举：`默认 / hover / active(按下或选中) / focus / disabled / loading / 空态(empty) / 错误态(error)`

## 一、响应式断点标准

### 1.1 断点定义（统一标准）

> 现有 HTML 高保真稿使用了 1100/900/768/640 等不一致的断点。**前端实现以本表为唯一标准**，覆盖旧稿中的散乱断点。

| 断点代号 | 宽度区间 | 列数 | 布局说明 |
|---------|---------|------|---------|
| **Desktop（桌面）** | `≥ 1280px` | 三列 | 左侧频道栏（260px）+ 主内容（1fr）+ 右侧栏（300–320px），完整布局 |
| **Tablet（平板）** | `768px – 1279px` | 两列 | 左侧栏 + 主内容；**隐藏右侧栏**，右侧栏内容（情绪看板/榜单）降级为主内容内的可折叠区块或顶部入口 |
| **Mobile（手机）** | `< 768px` | 单列 | 仅主内容；**左侧栏转为抽屉（Drawer）**，通过顶部导航汉堡菜单唤起；顶部导航折叠为汉堡菜单 |

对应 Tailwind 断点：`md = 768px`、`xl = 1280px`。默认样式写移动端（mobile-first），向上用 `md:` / `xl:` 覆盖。

### 1.2 断点通用行为

- **抽屉侧边栏（<768）**：点击汉堡菜单 → 抽屉从左侧滑入（`slide-right`，250ms `--ease-out`），叠加半透明遮罩层；点击遮罩、按 `Esc`、或选中一个频道后自动关闭。抽屉打开时 `body` 锁定滚动，焦点移入抽屉首个可聚焦元素（focus trap）。
- **右侧栏降级（768–1279）**：情绪看板改用 `EmotionBoard` 的 `compact` 模式，作为主内容顶部的可折叠面板；今日牛马榜降级为「查看完整榜单」入口链接。
- **触控目标**：<768 下所有可点击元素最小命中区域 44×44px。
- **横向溢出**：任何断点均不得出现横向滚动条（表格/榜单在 Mobile 下允许局部横滑并标注可滑动）。

### 1.3 各页面断点布局变化

| 页面 | Desktop ≥1280（三列） | Tablet 768–1279（两列） | Mobile <768（单列+抽屉） |
|------|----------------------|------------------------|--------------------------|
| **首页** `index.vue` | 频道侧边栏 + Hero/热门频道流 + 右侧栏（今日情绪看板 + 牛马榜） | 侧边栏 + 主流；右侧栏隐藏，情绪看板折叠进主流顶部，牛马榜转入口 | 抽屉侧边栏；Hero 压缩为单行标语；情绪看板/榜单堆叠为卡片纵向排列 |
| **频道页** `channel/[id].vue` | 频道侧边栏 + 消息流（含顶部情绪看板按钮）+ 线程面板（右侧内联滑出） | 侧边栏 + 消息流；线程面板改为**覆盖式浮层**从右滑入（fixed，宽度约 380px） | 抽屉侧边栏；消息流全宽；线程面板**全屏覆盖**从右滑入；情绪看板以底部 sheet/弹层呈现 |
| **日记广场** `diary/index.vue` (v1.5) | 侧边栏（分类+打卡日历）+ 瀑布流（masonry 3 列） | 侧边栏 + 瀑布流（2 列） | 抽屉侧边栏；瀑布流 1 列；打卡日历折叠进顶部可展开区 |
| **日记详情** `diary/[id].vue` (v1.5) | 居中内容列（≤760px）+ 右侧目录/作者卡（可选） | 居中内容列，右侧信息卡降级到内容底部 | 全宽内容列；评论区纵向堆叠；发评论输入框吸底 |
| **排行榜** `ranking.vue` | 领奖台（前三）+ 完整榜单表格 + 右侧筛选/说明 | 领奖台 + 榜单；右侧筛选转为榜单上方的横向 Tab | 领奖台横向压缩（头像缩小）；榜单转为卡片列表（隐藏次要列） |
| **个人主页** `profile/[id].vue` | 用户信息头 + 4 列统计卡 + 内容 Tab（消息/日记/共情） | 统计卡 2 列；内容 Tab 保留 | 统计卡 2 列；Tab 转为可横滑标签栏；内容单列 |

## 二、情绪系统交互

### 2.1 八种情绪：图标语义 + 色标建议

> `emoji` 列仅示意语义，UI 落地**禁用 emoji**，改用 SVG 线性图标（见 `design-system.md` 第七章）。色标为**建议值**，最终色板由 `claude-visual` 在 design-system 中统一定义并挂到 CSS 变量。

| key | 标签 | 语义 emoji | 图标语义（SVG 建议） | 建议色标（暗色 / 亮色语义） | 情感基调 |
|-----|------|-----------|---------------------|--------------------------|---------|
| `tired` | 疲惫 | 😮‍💨 | 呼气/云雾（`wind` / 叹气气泡） | 石板灰蓝 `#94a3b8` | 消耗、想休息 |
| `angry` | 愤怒 | 😡 | 火苗+皱眉（`flame` 变体，暖红） | 赤红 `#f87171` | 对不公的激烈情绪 |
| `wronged` | 委屈 | 😢 | 泪滴（`droplet`） | 忧郁蓝 `#60a5fa` | 憋屈、想哭 |
| `broken` | 崩溃 | 🤯 | 爆裂/裂纹（`zap` 破碎） | 品红紫 `#c084fc` | 情绪临界、失控 |
| `numb` | 麻木 | 😴 | 停滞/低电量（`battery-low` / `minus-circle`） | 冷灰 `#64748b` | 无感、机械化 |
| `escape` | 想润 | 🔥→✈ | 起飞/离开（`plane` / `door-open`） | 极光青 `#22d3ee` | 想逃离、换环境 |
| `anxious` | 焦虑 | 😰 | 波动/心跳（`activity` 波纹） | 琥珀黄 `#fbbf24` | 不安、担忧未来 |
| `cheer` | 加油 | 💪 | 上扬箭头/拳头（`trending-up` / `zap` 正向） | 共情绿 `#34d399` | 打气、正能量 |

- **色标使用规则**：情绪标签（`emo-tag`）用「色 @ 12% 透明底 + 25% 边 + 该色文字」；看板条形/环形用实色渐变。同一 key 在暗/亮主题下映射到各自 CSS 变量，保证对比度 ≥ 4.5:1（标签文字）。
- **无障碍**：情绪**绝不只靠颜色传达**——必须同时带图标 + 文字标签（色盲友好）。

### 2.2 情绪选择器 EmotionPicker 交互

组件结构见 `component-spec.md` §4。此处补齐**状态机、键盘导航、清除逻辑**。

**触发（Trigger）**
- 触发按钮（默认在 `MessageInput` 工具栏，文案「情绪」+ 图标）。点击 → 弹层开合切换（toggle）。
- 弹层定位：优先在触发按钮**上方**弹出（输入框通常在页面底部），带指向触发按钮的小箭头；自动计算避免溢出视口，空间不足时翻转到下方。

**状态**

| 状态 | 表现 | 说明 |
|------|------|------|
| 默认（未选） | 触发按钮显示「情绪」+ 图标，中性色 | 无选中 |
| 已选 | 触发按钮显示所选情绪图标+标签，底色染该情绪色标（12%） | `selected` 有值 |
| 弹层打开 | 触发按钮 `active`（高亮边框/微光），弹层 `fade` 淡入（150ms） | `isPanelOpen=true` |
| 选项 hover/focus | 选项浮起 2px + 该情绪色描边微光 | `--glow`（对应色） |
| 选项 selected | 选项实心该情绪色背景、勾选角标 | 单选高亮 |
| disabled | 触发按钮置灰、`cursor:not-allowed`、不可弹出 | 如未加入频道 |

**弹层内交互流程**
1. 8 个情绪以网格（建议 4×2）呈现，每项：图标 + 标签。
2. **单选（默认）**：点击某项 → 高亮 + `emit('select', { emotion: key })` + 关闭弹层。
3. **再次点击已选项 = 取消** → `emit('select', { emotion: null })`。
4. **清除选择**：已选状态下弹层底部出现「清除选择」按钮，点击 → `select(null)` 并关闭。
5. **多选模式**（`multiple=true`，预留）：选择不互斥，不自动关闭，`select` 返回数组。

**键盘可达性（A11y）**
- 触发按钮 `role="button"`，`aria-haspopup="listbox"`，`aria-expanded` 反映开合。
- 弹层 `role="listbox"`，选项 `role="option"` + `aria-selected`。
- 打开时焦点移入：已选项 → 该项；无选中 → 第一项。
- `←/→/↑/↓` 移动 `activeIndex`（网格环绕，行列均可移动）；`Home/End` 跳首尾。
- `Enter` / `Space` 确认当前 `activeIndex`；`Esc` 关闭并把焦点还给触发按钮；`Tab` 关闭弹层继续正常 Tab 流。
- 点击弹层外部（document click）关闭。

**反馈**
- 选中后触发按钮的染色过渡 200ms；已选情绪标签会跟随消息发送并在消息流中以 `emo-tag` 呈现。

### 2.3 情绪看板 EmotionBoard 交互

组件结构见 `component-spec.md` §5。此处补齐 **tab、tooltip、刷新、实时动效**。

**时间范围 Tab（today / week / all）**

| 状态 | 表现 |
|------|------|
| 默认 | 三个胶囊 Tab 横排，`today` 默认选中 |
| active（选中） | 选中 Tab 填充 AI 渐变/高亮底，文字反白 |
| hover/focus | 未选 Tab 底色微亮 + 下划线 |
| 切换中(loading) | 图表区骨架屏/半透明 + 顶部细进度条，Tab 可点但去抖 |

- **触发**：点击 Tab → `emit('change-range', { range })`，父组件按范围重新拉数。
- **A11y**：Tab 组用 `role="tablist"`，每个 `role="tab"` + `aria-selected`；`←/→` 在 Tab 间移动，`Enter/Space` 激活。

**Hover Tooltip（计数详情）**
- 触发：鼠标进入某条情绪（条形/环形段）→ 显示 tooltip：`{标签}：{count} 人（{percentage}%）`。
- 键盘等价：条目可 `Tab` 聚焦，`focus` 时同样显示 tooltip（`aria-describedby` 关联）。
- tooltip 跟随该条上方居中，避免遮挡；离开/失焦即隐藏（150ms 淡出）。

**刷新按钮**
- 触发：点击「刷新」→ `emit('refresh')`。
- 状态：`loading` 时图标旋转（`spin` 360° 循环）、按钮 disabled 防重复点击；完成后停转并给一次轻微高亮。
- A11y：按钮 `aria-label="刷新情绪看板"`，`aria-busy` 在加载时为 `true`。

**实时更新动效（WebSocket `emotion_update`）**
- 数据变化时条形宽度/环形弧度用 CSS `transition: width/stroke-dashoffset 600ms var(--ease-out)` 平滑过渡（不闪烁、不跳变）。
- 新增或跃升的情绪条给一次「光晕脉冲」（对应色 `--glow`，600ms 一次），提示「刚有人表达了情绪」。
- 计数数字用滚动/渐变过渡，避免生硬替换。
- **尊重减弱动效**：`@media (prefers-reduced-motion: reduce)` 时取消脉冲与滚动，仅做即时更新。

**状态汇总**

| 状态 | 表现 |
|------|------|
| 默认 | 条形图 + 图例 + 底部总计「共 N 位牛马表达了情绪」 |
| loading | 骨架条（占位灰块） |
| 空态 | 「还没有情绪数据，快来第一个发表心情吧」+ 引导发消息按钮 |
| 错误态 | 「情绪数据加载失败」+ 重试按钮（触发 refresh） |
| compact | 隐藏标题、缩小间距、仅前 N 条 + 「更多」；用于侧边栏/平板降级 |

## 三、核心组件交互状态与流程

### 3.1 频道侧边栏 ChannelSidebar

结构见 `component-spec.md` §7。补齐各元素状态与折叠/抽屉行为。

**搜索框**

| 状态 | 表现 / 行为 |
|------|-----------|
| 默认 | 占位「搜索频道...」，左侧 search 图标 |
| focus | 边框高亮（AI 色）+ 微光；出现清除「✕」按钮 |
| 输入中 | 实时 `emit('search', { keyword })`（防抖 200ms），频道列表即时过滤，空匹配的分类组隐藏 |
| 空结果 | 列表区显示「没有找到相关频道」+「创建同名频道」快捷入口 |
| 清除 | 点击「✕」或 `Esc` 清空并恢复完整列表 |
| A11y | `role="searchbox"`，`aria-label="搜索频道"`；`Esc` 清空 |

**分类折叠箭头（category-header）**

- 触发：点击分类标题（行业/岗位/主题/自建）→ 切换展开/收起，箭头 `▶`（收起）↔ `▼`（展开）旋转 90°（200ms）。
- 状态存于 `expandedCategories`；默认全部展开。
- A11y：`aria-expanded` + `aria-controls` 指向该组列表；`Enter/Space` 触发。

**未读徽章 unread-badge**

| 状态 | 表现 |
|------|------|
| 无未读 | 不显示徽章 |
| 有未读 | 频道名右侧红色胶囊数字；`>99` 显示「99+」 |
| 折叠态 | 徽章缩为图标右上角的小红点（dot，不显示数字） |
| 进入该频道后 | 徽章清零并淡出 |
| A11y | 徽章 `aria-label="N 条未读"`，随消息实时更新 |

**频道项状态**

| 状态 | 表现 |
|------|------|
| 默认（未加入） | 图标+名称，中性色 |
| joined（已加入） | 高亮/置顶显示 |
| active（当前频道） | 左侧强调条 + 高亮底 |
| hover/focus | 底色微亮、浮起 |
| 折叠态 hover | 显示频道名 tooltip |

**折叠态（collapsed，仅 Desktop/Tablet）**
- 触发：点击折叠按钮 → 侧边栏从 260px 收窄到 72px（250ms `--ease-out`），仅显示图标；`emit('toggle-collapse')`。
- 折叠时：隐藏搜索框、分类文字、频道名、未读数字（红点保留），hover 图标出 tooltip。
- 状态持久化到 localStorage。
- **注意**：<768（Mobile）不使用 collapsed，而是整体转为**抽屉**（见 §1.2）。

**创建频道按钮**
- 触发：点击底部「＋ 创建频道」→ `emit('create-channel')`，父组件弹出创建对话框（模态）。
- 状态：hover 浮起 + 微光；未登录时置灰，点击引导登录。
- 折叠态：只显示「＋」图标。
- 模态 A11y：打开时 focus trap，`Esc` 关闭，关闭后焦点还给该按钮。

### 3.2 顶部导航 TopNav

结构见 `component-spec.md` §8。补齐下拉面板、等级徽章、汉堡菜单。

**通知下拉面板**

| 状态 | 表现 / 行为 |
|------|-----------|
| 默认 | 铃铛图标；有未读时右上角红色 badge（数字，`>99` 显示「99+」） |
| 点击展开 | `dropdown` 过渡展开面板，顶部「通知」标题 +「全部已读」 |
| 列表项-未读 | 左侧色点 + 加粗；点击 → `mark-notification-read` + 跳转关联页（refId），并标记已读 |
| 列表项-已读 | 常规样式 |
| 按类型图标 | mention=@ / empathy=共情图标 / reply=回复图标 / system=系统图标 |
| loading | 面板内骨架条 |
| 空态 | 「暂无通知，安心搬砖」插画+文案 |
| 错误态 | 「通知加载失败」+ 重试 |
| 全部已读 | 点击 → `mark-all-read`，所有项去除未读态，badge 清零 |
| 关闭 | 点击外部 / `Esc` / 再次点击铃铛 |
| A11y | 铃铛 `aria-haspopup="menu"` + `aria-expanded`；面板 focus trap；badge 用 `aria-label="N 条未读通知"` |

**用户下拉菜单**
- 触发：点击头像区 → 展开菜单（个人主页 / 我的日记 / 设置 / 退出登录）。
- 状态：hover 项高亮；「退出登录」用 danger 色。
- 关闭：点击外部 / `Esc` / 选中一项后。
- A11y：`role="menu"`，项 `role="menuitem"`，`↑/↓` 导航，`Enter` 激活，`Esc` 关闭回焦头像。

**Lv. 牛马等级徽章**
- 位置：头像旁，显示 `Lv.{level}`。
- 视觉分级建议（供 visual 落地）：低阶中性灰 → 中阶 AI 蓝 → 高阶暖金/渐变，等级越高越醒目，增强归属与成就感。
- hover：tooltip 显示「牛马等级 · {等级名} · 距下一级还差 N 共情」。
- A11y：`aria-label="牛马等级 {level}"`，纯装饰的等级光效标 `aria-hidden`。

**移动端汉堡菜单（<768）**
- 触发：点击汉堡 → `slide-down` 展开垂直导航（首页/频道/日记广场/排行榜）+ 搜索入口 + 用户/登录入口。
- 同时承担唤起频道抽屉的入口（在频道页语境下）。
- 状态：展开时汉堡图标转为「✕」；选中任一项后自动收起；遮罩点击关闭。
- A11y：`aria-expanded`、`aria-controls`；展开时 focus trap，`Esc` 关闭。

**搜索框（导航）**
- Desktop 常驻，<768 收进汉堡菜单。`Enter` 提交 → `emit('search')` 跳搜索结果页；空关键词不提交。

### 3.3 共情按钮 EmpathyButton（核心特色）

结构见 `component-spec.md` §6。补齐动画反馈、乐观更新、实时同步的状态语义。

**状态**

| 状态 | 表现 |
|------|------|
| 未共情（默认） | 图标（拥抱）+「我懂你」+ 计数；共情色描边、软底（`empathy-soft`） |
| hover/focus | 放大 1.05 + `--glow-empathy`，文案不变 |
| active（已共情） | 实心共情渐变底、深色文字「已懂你」；计数含本次 |
| animating | 点击瞬间图标弹跳 + 6–8 个心形/拥抱粒子向外扩散（600ms），结束 `emit('animation-end')` |
| pulse（他人共情） | 收到他人 WebSocket `empathy` 时，播放一次轻微脉冲环，不改变自身按钮 active 态 |
| disabled | 未登录置灰，点击 → 引导登录页（不发请求） |
| loading（去抖窗口） | 视觉保持乐观态，按钮短暂不可再次触发 |
| error（回滚） | API 失败：`localCount`/`localEmpathized` 回滚，轻提示 toast「操作失败，请重试」 |

**点击流程（抱团取暖）**
1. 点击 → **乐观更新**：立即切换 `localEmpathized` 并 `localCount ±1`，播放弹跳+粒子动画。
2. `emit('empathy', { messageId, action: 'add' | 'remove' })`。
3. 成功：保持；失败：回滚 + toast。
4. **防抖**：500ms 内重复点击只发一次请求（净结果生效）。
5. 尺寸：`sm`=线程回复、`md`=消息列表、`lg`=日记详情。

**取消共情**
- 已 active 时再次点击 → `action:'remove'`，图标回落（无粒子扩散，或反向收拢动画），计数 -1，回未共情态。

**实时同步（WebSocket `empathy`）**
- 他人共情同一条消息 → `count` Prop 更新，若**非本人操作**则播放 `pulse-ring`（提示「有人也懂你了」），本人 active 态不误翻转。

**A11y**
- `role="button"`，`aria-pressed` = 是否已共情；`aria-label="我懂你，当前 N 人共情"` 随状态更新。
- 粒子/脉冲为装饰，`aria-hidden`；`prefers-reduced-motion` 下关闭粒子与脉冲，仅计数变化。
- `Enter/Space` 等价点击。

### 3.4 线程面板 ThreadPanel

结构见 `component-spec.md` §3。补齐打开/关闭与回复流状态。

**打开/关闭**
- 触发打开：消息项点击「N 回复」→ 父组件设 `visible=true`。
- 呈现：Desktop 右侧**内联滑出**；Tablet **fixed 覆盖式**从右滑入（约 380px）+ 遮罩；Mobile **全屏覆盖**从右滑入（`slide-right`，250ms）。
- 关闭：点击「✕」/ 点击遮罩 / `Esc` → `emit('close')`；关闭保留滚动位置，再次打开恢复。
- A11y：打开时 `role="dialog"` + `aria-modal`（覆盖式时），focus trap，焦点移入面板；关闭回焦到来源消息。

**回复流状态**

| 状态 | 表现 |
|------|------|
| loading | 回复列表区「加载中...」骨架 |
| 空态 | 父消息下方「还没有人回复，来接住 TA」+ 输入框聚焦引导 |
| 列表 | 父消息固定在顶部区，下方「N 条回复」分隔，回复项含共情按钮（`sm`） |
| 实时追加 | 收到 `thread_reply` → 追加并自动滚到底部（若用户已在底部） |
| 发送中 | 回复按钮 loading、输入禁用；乐观插入本地回复占位 |
| 错误态 | 发送失败：占位标红 + 重试入口 |

**发送回复**
- `Enter` 发送（`Shift+Enter` 换行）或点按钮 → `emit('send-reply', { parentId, content, isAnonymous })`；成功清空输入框。
- 空内容禁用发送按钮。匿名开关逻辑同 MessageInput。

### 3.5 消息输入框 MessageInput

结构见 `component-spec.md` §2。补齐工具项与发送态。

**表情**
- 触发：点击表情按钮 → 展开表情快捷面板（网格），点击插入到光标位置。
- 面板 `Esc`/外部点击关闭；A11y：`role="dialog"`/`listbox`，箭头导航 + `Enter` 插入。
- 注意：表情是**输入内容里的字符**，与「情绪标签（EmotionPicker）」是两件事，不要混淆。

**情绪标签**
- 触发：点击「情绪」→ 打开 `EmotionPicker`（见 §2.2）；选中后工具栏显示已选标签，可点标签上的「✕」或再次进 picker 清除。

**匿名开关**

| 状态 | 表现 |
|------|------|
| 关（默认） | 开关灰态，正常署名发送 |
| 开 | 开关高亮（`eye-off` 图标），提示「本条将以『匿名牛马』发送」 |
| 切换 | `emit('toggle-anonymous', { isAnonymous })`；该状态**发送后保留**（连续匿名吐槽场景） |
| disabled | `allowAnonymous=false` 时隐藏开关 |
| A11y | `role="switch"` + `aria-checked`；`Space` 切换 |

**发送态**

| 状态 | 表现 / 行为 |
|------|-----------|
| 默认 | 发送按钮可用性取决于 `content.trim()` 非空 |
| 不可发送 | 内容为空 / 超字数 → 按钮 disabled |
| 字数 | `{len}/{max}` 计数；接近上限变黄，超限变红并阻止继续输入 |
| 发送中(loading) | 按钮转圈 + disabled，防重复提交 |
| 发送成功 | 清空 `content`/`selectedEmotion`/`attachments`，保留 `isAnonymous`；输入框重新聚焦 |
| 发送失败(error) | 保留输入内容 + toast「发送失败，点击重试」 |
| disabled（整框） | 未加入频道时禁用，显示「加入频道后即可发言」引导 |
| typing | 开始输入触发 `typing(true)`，停止 2s 触发 `typing(false)`（防抖，供 WS 广播） |

**附件**
- 点击上传 → 选文件，图片生成缩略图预览，可删除；受类型（image/document）与大小（≤10MB）限制，超限提示。
- A11y：预览项删除按钮 `aria-label="移除附件 {文件名}"`。

**快捷键汇总**：`Enter` 发送、`Shift+Enter` 换行、`Esc` 关闭当前弹层（表情/情绪）。

## 四、日记详情页交互流程（v1.5）

> **【v1.5 功能】** 架构师裁定「打工日记」为 v1.5 交付，v1.0 不实现。本节为前瞻规范，实现顺序以架构文档为准。对应 `pages/diary/[id].vue`。

### 4.1 页面构成与布局

- **内容区**：标题 + 作者卡（头像/昵称/牛马等级/发布时间）+ 心情标签（复用情绪色标语义）+ 正文（富文本/Markdown 渲染）+ 连续打卡天数徽章。
- **交互操作条**：共情按钮（`lg` 尺寸，见 §3.3）+ 收藏 + 分享 + （作者本人）编辑/删除。
- **评论区**：评论输入 + 评论列表。
- 布局断点见 §1.3：Desktop 居中内容列（≤760px）+ 可选右侧作者/目录卡；Mobile 全宽 + 发评论输入吸底。

### 4.2 内容展示状态

| 状态 | 表现 |
|------|------|
| loading | 标题/正文骨架屏 |
| 正常 | 完整渲染；图片懒加载，点击可放大预览 |
| 私密日记（非作者） | 「这是一篇私密日记」占位，不展示正文 |
| 404/已删除 | 「日记不存在或已被删除」+ 返回日记广场 |
| 错误态 | 「加载失败」+ 重试 |

### 4.3 评论区交互流程

**发评论**

| 状态 | 表现 / 行为 |
|------|-----------|
| 默认 | 底部评论输入框，占位「说点什么安慰一下 TA...」，可选匿名开关 |
| 未登录 | 输入框置灰，点击引导登录 |
| 输入中 | 字数计数；`Enter` 发送 / `Shift+Enter` 换行 |
| 发送中 | 按钮 loading + disabled；乐观插入本地评论占位 |
| 成功 | 清空输入、新评论出现在列表（顶部或底部，按排序），评论计数 +1 |
| 失败 | 占位标红 + 重试；输入内容保留 |
| A11y | 输入区 `aria-label="发表评论"`；发送 `Enter` 可达 |

**评论列表**

| 状态 | 表现 |
|------|------|
| loading | 骨架条 |
| 空态 | 「还没有评论，来当第一个安慰 TA 的人」 |
| 列表项 | 头像 + 昵称（或「匿名牛马」）+ 内容 + 时间 + 该评论的共情/回复入口 |
| 分页/加载更多 | 底部「加载更多评论」或滚动触底自动加载（loading 指示） |
| 作者本人评论 | 提供删除入口（确认二次弹窗） |
| 错误态 | 「评论加载失败」+ 重试 |

**流程要点**
- 匿名评论不暴露 `user_id`（与消息匿名规则一致）。
- 评论支持共情（复用 EmpathyButton）。
- 删除评论：点击删除 → 确认对话框 → 成功后列表移除 + 计数 -1。

## 五、全局交互约定与可达性基线

> 所有组件默认遵守，本节不再逐组件重复。

### 5.1 焦点与键盘

- **可见焦点环**：所有可交互元素 `:focus-visible` 显示清晰焦点环（AI 色 2px），键盘用户可见、鼠标点击不打扰。
- **Tab 顺序**：符合视觉阅读顺序（左→右、上→下）；跳过装饰元素。
- **弹层/模态/抽屉/线程面板**：打开时 focus trap（焦点锁在内部），`Esc` 关闭并把焦点还给触发源。
- **跳转链接**：页面顶部提供「跳到主内容」skip link（键盘 Tab 首个可聚焦）。

### 5.2 状态一致性

- **loading**：优先骨架屏（保持布局稳定），按钮内 loading 用 spinner + disabled 防重复提交。
- **空态**：文案贴合「牛马」语气 + 明确的下一步引导按钮，不留白屏。
- **错误态**：可读文案 + 重试入口，禁止只弹一句技术报错。
- **乐观更新**：共情、评论、发消息采用乐观更新，失败回滚 + toast。

### 5.3 动效与减弱

- 时长/缓动遵循 `design-system.md` §6（150/250/400/600ms；`--ease-out/-in-out/-spring`）。
- **`@media (prefers-reduced-motion: reduce)`**：关闭粒子、脉冲、流动渐变、大幅位移，保留必要的即时状态切换（透明度/颜色）。

### 5.4 触控与命中区

- 移动端可点元素最小 44×44px；相邻可点元素间距 ≥ 8px，避免误触。
- 长列表（消息流/榜单/评论）滚动流畅，滚动位置在分页加载后不跳变。

### 5.5 隐私与匿名（交互层面）

- 匿名消息/评论一律显示「匿名牛马」+ 默认匿名头像，**不渲染任何指向真实用户的链接/跳转**。
- 匿名开关状态对用户可见、可预期；发送前给「本条将匿名」的明确提示。
