## 一、产品定位与价值主张

### 1.1 产品定位

**Alike** 是一个面向中国广大打工人（自嘲为"牛马"）的**情感共鸣型 Web 聊天社区**。它不是另一个工作效率工具，也不是一个冰冷的信息流平台——它是一个让疲惫的灵魂互相看见、互相理解、互相取暖的数字避风港。

在当下的中国职场环境中，"牛马"一词已成为数亿打工人的集体自嘲符号。从 996 的互联网程序员到风雨无阻的外卖骑手，从流水线上的产业工人到三甲医院里连轴转的护士，几乎每个打工人都在经历着不同程度的职业倦怠、情绪压抑和社交孤立。然而，现有的社区产品要么过于泛化（贴吧、微博），要么过于工具化（Slack、钉钉），要么过于焦虑制造导向（脉脉、知乎），缺乏一个真正以**情绪共鸣**为核心诉求的垂直社区。

Alike 填补的正是这一空白。通过频道（Channel）组织同行业、同岗位的牛马群体，通过线程（Thread）支持深度讨论与互助问答，通过情绪标签和情绪看板让集体情绪可视化，通过"抱团取暖"（共情按钮）替代传统的"点赞"来传递真正的人际温暖，Alike 致力于成为打工人的精神自留地。

### 1.2 核心价值主张

| 价值维度 | 主张 | 说明 |
|----------|------|------|
| **共鸣** | "总有人懂你的辛苦" | 你不是一个人在扛。在 Alike，相似经历的牛马们会理解你的疲惫、愤怒和委屈。情绪标签让每一句吐槽都有回应，抱团取暖让每一次倾诉都有温度。 |
| **归属** | "找到你的同路人" | 按行业/岗位/主题精准分频道，三秒找到自己人。#互联网、#工厂、#外卖骑手、#医护、#教培——不管你在哪个坑里，这里都有同坑的兄弟姐妹。 |
| **宣泄** | "健康释放，不压抑" | 匿名吐槽机制让你放下包袱畅所欲言，情绪标签帮你给压力命名，打工日记给你一个倾诉的出口。不是抱怨，是释放；不是消极，是自救。 |
| **互助** | "经验共享，少走弯路" | Slack 式线程讨论支持深度问答，薪资揭秘频道帮你了解行业行情，维权互助频道帮你保护自身权益。牛马帮牛马，才是真正的团结。 |
| **陪伴** | "你不是孤岛" | 打工日记打卡日历记录你每天的坚持，日记广场让你的心路被看见，连续打卡徽章给你小小的成就感。有人在看，有人在等，这就是陪伴。 |

### 1.3 产品愿景与使命

**愿景：** 成为中国打工人的第一情感社区，让每一个辛苦的灵魂都能找到归属。

**使命：** 用技术连接相似的境遇，用共鸣化解职场的孤独，让"牛马"不再是一个人的战斗。

### 1.4 产品差异化

与现有社区产品的核心差异在于——Alike 是**以情绪为第一公民**的社区设计：

1. **情绪标签是消息的必选属性**：发消息时选择当前情绪状态，让接收者不只看到文字，更感受到情绪
2. **情绪看板实时映射集体心理**：频道级的情绪热力图，一眼看到今天大家是什么状态
3. **"抱团取暖"替代"点赞"**：共情不是社交媒体式的浅层认可，而是"我懂你"的深层情感连接
4. **牛马等级体系激励互助**：等级不只看活跃度，更看被共情次数——你帮助了多少人，你就有多高的地位
5. **打工日记形成纵向陪伴**：不只是即时聊天，日记系统提供反思与记录的慢节奏出口

## 二、目标用户画像

### 2.1 用户群体概述

Alike 的目标用户是**中国 18-45 岁的在职打工人**，覆盖白领、蓝领、服务业从业者、灵活就业者等全光谱打工群体。他们共同的特征是：在工作中承受着不同程度的压力，需要一个安全的出口来释放情绪、获取共鸣和寻求帮助。

## 三、核心功能需求列表

### 3.1 优先级定义

| 优先级 | 定义 | 说明 |
|--------|------|------|
| **P0** | 必须有 (Must Have) | MVP 上线的必要功能，缺一不可 |

### 3.2 功能需求总表

#### P0 — 必须有（MVP 核心功能）

| 编号 | 功能模块 | 功能名称 | 描述 | 验收标准 |
|------|----------|----------|------|----------|
| P0-01 | 用户系统 | 注册与登录 | 邮箱注册，密码登录，JWT 认证 | 用户可通过邮箱+密码完成注册和登录，获取 JWT Token |
| P0-02 | 用户系统 | 个人主页 | 展示用户基本信息：昵称、头像、行业、岗位、工龄、个人简介、牛马等级 | 用户可查看自己和他人主页，可编辑自己的资料（含个人简介，≤200 字符） |
| P0-03 | 用户系统 | 匿名模式 | 发消息/日记时可选择匿名，隐藏真实身份 | 匿名消息不显示用户真实信息，显示统一匿名标识 |
| P0-04 | 用户系统 | 牛马等级 | 根据活跃度（发帖数、登录天数）和被共情次数综合计算等级 | 等级分为：小牛马(Lv1)→初级牛马(Lv2)→中级牛马(Lv3)→老牛马(Lv4)→牛马之王(Lv5)，等级显示在用户名片上 |
| P0-05 | 频道系统 | 频道列表与分类 | 频道按 industry/job/topic 三类分组展示，支持搜索 | 首页左侧边栏展示所有频道，按分类分组，可搜索 |
| P0-06 | 频道系统 | 加入/离开频道 | 用户可自由加入或离开公开频道 | 加入后出现在"我的频道"列表，离开后不再接收该频道消息 |
| P0-07 | 频道系统 | 预置频道 | 系统预置至少 16 个频道覆盖主流行业和热门主题 | 预置频道包括：#互联网、#工厂、#外卖骑手、#医护、#教培、#服务业、#程序员、#产品经理、#设计师、#流水线、#骑手、#吐槽大会、#打工日记、#薪资揭秘、#摸鱼技巧、#维权互助 |
| P0-08 | 消息系统 | 发布消息 | 支持文本消息发布，支持选择情绪标签 | 消息成功发布后实时显示在频道消息流中 |
| P0-09 | 消息系统 | 消息历史与分页 | 频道消息按时间倒序排列，支持向上滚动加载历史消息 | 默认加载最近 50 条，每次上拉加载 50 条，无重复无遗漏 |
| P0-10 | 消息系统 | 实时消息推送 | WebSocket 实时推送新消息到频道在线成员 | 新消息延迟 < 2 秒到达所有在线成员客户端 |
| P0-11 | 消息系统 | 线程回复 | 在任意消息下可展开线程进行回复讨论，类似 Slack | 点击消息可展开线程面板，显示所有回复，支持在面板内继续回复 |
| P0-12 | 情绪系统 | 情绪标签 | 发消息时从预设情绪标签中选择一个：😮‍💨疲惫、😡愤怒、😢委屈、🤯崩溃、😴麻木、🔥想润、😰焦虑、💪加油 | 每条消息可关联一个情绪标签，标签以 emoji+文字形式展示在消息上 |
| P0-13 | 情绪系统 | 情绪看板 | 频道页展示实时情绪分布热力图，显示当前频道各情绪标签的占比 | 以可视化图表（柱状图或热力图）展示频道内近 24 小时的情绪分布，每 30 秒刷新一次 |
| P0-14 | 共情机制 | 抱团取暖（共情按钮） | 每条消息下方有"抱团取暖"按钮，点击后共情数+1，再次点击取消 | 每人每条消息只能共情一次，共情后按钮高亮显示共情数，取消后恢复 |
| P0-15 | 共情机制 | 共情实时通知 | 被共情的用户收到实时通知 | WebSocket 推送共情事件，通知栏显示"xxx 给你的消息抱团取暖了" |
| P0-16 | 通知系统 | 通知列表 | 展示所有通知（共情、回复、@提及、系统通知），支持已读/未读标记 | 通知按时间倒序排列，未读通知有标记，可标记单条已读或全部已读 |
| P0-17 | 通知系统 | @提及通知 | 消息中输入 @ 可触发用户搜索，选择后对方收到提及通知 | 支持 @昵称 格式，被提及者收到通知并可在通知中跳转到对应消息 |
| P0-18 | 通知系统 | 回复通知 | 消息被线程回复时，原消息作者收到通知 | 通知包含回复者信息、回复内容摘要、跳转链接 |
| P0-19 | 首页 | 首页热门内容 | 展示热门频道、今日最受共情帖子（今日牛马榜） | 首页展示频道分类入口 + 今日 Top 10 共情帖子列表 |
| P0-20 | 消息系统 | 删除自己的消息 | 用户可删除自己发布的消息 | 删除后消息从列表消失，线程回复一并删除，其他用户看到"消息已删除"占位 |

#### P1 — 应该有（重要迭代功能）

### 4.2 安全需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-S01 | 密码安全 | 用户密码使用 bcrypt 加盐哈希存储，禁止明文存储或传输。密码强度要求：至少 8 位，包含字母和数字 |
| NFR-S02 | JWT 认证 | 所有 API 请求需携带有效 JWT Token（登录/注册接口除外）。Token 有效期 2 小时，Refresh Token 有效期 7 天 |
| NFR-S03 | HTTPS 强制 | 全站 HTTPS，WebSocket 使用 WSS 协议，禁止 HTTP 明文传输 |
| NFR-S04 | SQL 注入防护 | 使用参数化查询（pgx 预编译语句），禁止字符串拼接 SQL |
| NFR-S05 | XSS 防护 | 前端输出 HTML 转义用户输入内容，禁止使用 v-html 直接渲染用户内容 |
| NFR-S06 | CSRF 防护 | JWT 方案天然防 CSRF，但敏感操作（删除、修改）需二次确认 |
| NFR-S07 | 限流防刷 | API 限流：单 IP 每分钟 ≤ 60 次请求；发消息限流：单用户每分钟 ≤ 20 条；日记限流：单用户每天 ≤ 10 篇 |
| NFR-S08 | 文件上传安全 | 限制文件类型（image/jpeg, image/png, image/gif, image/webp, application/pdf, text/plain），限制大小（图片 ≤ 5MB，文档 ≤ 10MB），文件名随机化存储 |
| NFR-S09 | 匿名隐私保护 | 匿名消息在数据库中仍记录真实 user_id（用于内容审核），但 API 返回时不包含任何可识别信息，匿名身份不可逆向追踪 |
| NFR-S10 | 敏感信息保护 | 日志中不记录用户密码、Token 等敏感信息；错误响应不暴露系统内部结构（如 SQL 错误、堆栈信息） |
| NFR-S11 | 数据备份 | PostgreSQL 每日自动备份，保留 30 天；MinIO 文件每周备份 |
| NFR-S12 | 内容审核 | 建立关键词过滤机制（政治敏感、色情、暴力等），违规内容自动标记并通知管理员审核 |

### 5.3 竞品深度分析

#### 5.3.1 Discord — 组织结构参考，情绪维度缺失

**可借鉴之处：**
- **频道+线程的内容组织模式**：服务器→频道→线程的层级结构非常适合社区讨论，Alike 直接采用频道→线程的模式
- **实时消息体验**：WebSocket 推送、输入中提示、在线状态等实时交互设计值得借鉴
- **表情反应（Reactions）**：轻量级的情绪表达机制，Alike 将其升级为"情绪标签"（每条消息必选一个情绪）和"抱团取暖"（共情按钮）

**Alike 的差异化：**
- Discord 是兴趣驱动的，Alike 是**境遇驱动**的——用户因共同的打工身份而聚集，而非共同爱好
- Discord 缺乏情绪可视化工具，Alike 的**情绪看板**让集体情绪实时可见
- Discord 的表情反应是附加的、可选的，Alike 的情绪标签是**消息的一等属性**，强制让用户表达情绪状态
- Discord 面向全球游戏玩家，Alike 聚焦**中国职场打工人**这一特定人群

#### 5.3.2 Slack — 线程模式参考，场景完全不同

**可借鉴之处：**
- **线程回复（Threads）的交互设计**：在消息下方展开线程面板，不干扰主消息流，Alike 直接采用此模式
- **@提及和通知系统**：精确的用户提及和通知分类管理
- **频道分类管理**：频道可按主题分组展示

**Alike 的差异化：**
- Slack 是工作场景工具，本身是压力的**来源**之一；Alike 是压力的**出口**
- Slack 的频道是组织内部的，Alike 的频道是**跨组织**的——不同公司的程序员在同一个 #程序员 频道交流
- Slack 没有情感化设计，Alike 以情绪为核心设计语言
- Slack 是 B2B SaaS，Alike 是 **C 端社区**

#### 5.3.3 百度贴吧 — 用户基数参考，体验严重过时

**可借鉴之处：**
- **主题吧的分类逻辑**：按兴趣/行业/地区分类的思路，Alike 转化为行业/岗位/主题频道
- **帖子+楼中楼的深度讨论**：贴吧的楼中楼是早期线程讨论的形式，Alike 用 Slack 式线程替代
- **匿名发帖传统**：贴吧的匿名文化降低了表达门槛，Alike 保留匿名模式

**Alike 的差异化：**
- 贴吧是异步的帖子论坛，Alike 是**实时聊天社区**——消息即时推送，讨论即时进行
- 贴吧内容泛化、水化严重，Alike 聚焦**打工主题**，内容垂直度高
- 贴吧缺乏情绪化设计，Alike 的情绪标签和看板是核心差异
- 贴吧的"赞"是社交媒体式的浅层认可，Alike 的"抱团取暖"传递的是**深层共情**
- 贴吧体验陈旧（PC 时代设计），Alike 采用**现代技术栈**（Vue3+Nuxt3 SSR），体验对标 Slack/Discord

#### 5.3.4 即刻 — 社区氛围参考，用户群体不同

**可借鉴之处：**
- **圈子（社区）的组织方式**：按兴趣/主题分圈子，Alike 类似但按行业/岗位/主题分频道
- **社区氛围营造**：即刻营造了一种"懂你"的社区氛围，Alike 将这种感觉做到极致
- **动态流的内容消费模式**：首页展示精选内容，Alike 的首页展示今日牛马榜（最受共情帖子）

**Alike 的差异化：**
- 即刻的用户偏一二线城市互联网从业者，Alike 面向**全行业全城市**的打工人，包括蓝领和服务业
- 即刻是动态流模式（异步），Alike 是**实时聊天**模式（同步），互动即时性更强
- 即刻没有情绪系统，Alike 的情绪标签和情绪看板是核心特色
- 即刻的认可是"点赞"，Alike 是"抱团取暖"——从认可到共情的语义升级
- 即刻偏精英社区调性，Alike 是**草根共鸣**调性

### 5.4 竞品分析总结：Alike 的独特价值

经过以上分析，Alike 在竞品格局中的独特定位可以概括为：

> **Alike = Discord 的频道+线程结构 × 贴吧的草根匿名文化 × 即刻的社区温度 × 独创的情绪系统与共情机制**

具体差异化优势：

1. **情绪是一等公民**：唯一将情绪标签作为消息必选属性、提供频道级情绪看板的社区产品
2. **共情替代点赞**："抱团取暖"不只是认可，是"我懂你"的情感传递，语义层级高于点赞/赞
3. **打工人垂直定位**：不泛化、不精英化，真正面向全行业打工人的草根社区
4. **匿名+实时结合**：既有贴吧式的匿名自由，又有 Slack/Discord 式的实时互动
5. **打工日记沉淀**：不只是即时聊天，日记系统提供了反思和纵向陪伴的慢节奏出口

### 6.2 MVP v1.0 功能边界

#### 6.2.1 包含的功能（P0 全量）

MVP v1.0 包含所有 P0 优先级功能（P0-01 至 P0-20），即：

| 模块 | 功能 |
|------|------|
| 用户系统 | 注册登录、个人主页、匿名模式、牛马等级（基础版） |
| 频道系统 | 频道列表分类、加入/离开频道、16 个预置频道 |
| 消息系统 | 发布消息、消息历史分页、实时推送、线程回复、删除消息 |
| 情绪系统 | 情绪标签（8 种）、情绪看板（实时热力图） |
| 共情机制 | 抱团取暖按钮、共情实时通知 |
| 通知系统 | 通知列表（共情/回复/@提及/系统）、已读/未读管理 |
| 首页 | 热门频道入口 + 今日牛马榜（Top 10 共情帖子） |

#### 6.2.2 明确不包含的功能（v1.0 边界外）

以下功能在 v1.0 中**明确不做**，留待后续版本：

| 功能 | 留待版本 | 原因 |
|------|----------|------|
| 打工日记（全部） | v1.5 | 日记系统是独立的慢节奏功能，MVP 聚焦实时聊天核心体验 |
| 排行榜页面 | v1.5 | 依赖共情数据积累，MVP 阶段数据量不足以支撑有意义的排行 |
| 文件/图片上传 | v1.5 | MinIO 集成和文件管理增加复杂度，MVP 先支持纯文本+表情 |
| 全文搜索 | v1.5 | 需要 Elasticsearch 或 PostgreSQL 全文索引优化，MVP 数据量不大时可用简单 LIKE 查询过渡 |
| 自建频道 | v2.0 | 需要审核流程和管理后台，MVP 使用预置频道即可 |
| 消息编辑 | v2.0 | 增加编辑历史管理复杂度，MVP 支持删除即可 |
| 频道管理员工具 | v2.0 | 需要管理后台，MVP 阶段由系统管理员直接操作数据库 |
| 举报与审核系统 | v2.0 | 需要审核后台和处理流程，MVP 阶段靠人工监控 |
| 关注用户 | v2.0 | 社交关系链是进阶功能，MVP 聚焦频道内的公开交流 |
| 个人数据报告 | v2.0 | 需要数据积累和分析能力 |
| 暗色/亮色切换 | v1.5 | MVP 默认暗色主题，后续增加切换 |
| 通知偏好设置 | v2.0 | MVP 阶段所有通知默认开启 |
| 频道推荐 | v2.0 | 需要推荐算法和用户行为数据 |
| 个人情绪趋势 | v1.5 | 依赖个人消息数据积累 |

#### 6.2.3 MVP 验收标准

MVP v1.0 上线需满足以下全部条件：

**功能完整性：**
- [ ] 所有 P0 功能（P0-01 至 P0-20）开发完成并通过测试
- [ ] 用户可完成注册→登录→加入频道→发消息→选择情绪→收到回复→收到共情→收到通知的完整流程
- [ ] WebSocket 实时推送功能正常（新消息、共情、通知）
- [ ] 情绪看板实时展示频道情绪分布

**性能达标：**
- [ ] API P95 响应时间 < 200ms
- [ ] 消息推送延迟 < 2 秒
- [ ] 首页首屏加载 < 3 秒
- [ ] 支持 100 并发用户压测通过（内部测试标准）

**安全达标：**
- [ ] HTTPS + WSS 全站加密
- [ ] JWT 认证全链路生效
- [ ] 密码 bcrypt 加密存储
- [ ] API 限流生效
- [ ] XSS/SQL 注入防护验证通过

**可用性达标：**
- [ ] PC + 移动端响应式适配完成
- [ ] 暗色主题视觉完整（Aurora 极光风格）
- [ ] 断线重连机制正常工作
- [ ] 离线消息不丢失

**部署达标：**
- [ ] Docker Compose 一键启动开发环境
- [ ] 数据库迁移脚本可执行
- [ ] 种子数据可一键灌入（至少 16 频道 + 50 用户 + 200 消息）

## 七、附录

### 7.1 术语表

| 术语 | 定义 |
|------|------|
| 牛马 | 打工人的自嘲称呼，本产品中泛指所有在职打工人 |
| 频道 (Channel) | 按行业/岗位/主题组织的聊天室，类似 Discord/Slack 的 Channel |
| 线程 (Thread) | 在某条消息下展开的回复讨论串，不干扰主消息流 |
| 情绪标签 (Emotion Tag) | 发消息时选择的情绪状态标签，如"疲惫""愤怒""崩溃"等 |
| 情绪看板 (Emotion Board) | 频道内实时情绪分布的可视化图表 |
| 抱团取暖 (Empathy) | Alike 的核心共情机制，类似"点赞"但语义为"我懂你" |
| 牛马等级 (Level) | 根据用户活跃度和被共情次数计算的用户等级 |
| 打工日记 (Diary) | 用户记录每日打工感受的日志，可公开或私密 |
| 日记广场 (Diary Plaza) | 公开打工日记的聚合展示页面 |
| 打卡日历 (Streak Calendar) | 记录用户连续写日记天数的日历视图 |
| 今日牛马榜 | 首页展示的当日最受共情帖子 Top 10 |

### 7.2 情绪标签定义

| 标签 | Emoji | 语义 | 使用场景 |
|------|-------|------|----------|
| 疲惫 | 😮‍💨 | 身体或精神上的疲劳感 | 加班后、长时间工作后 |
| 愤怒 | 😡 | 对不公正待遇或不合理要求的愤怒 | 被领导PUA、被甲方刁难 |
| 委屈 | 😢 | 被误解或不被认可的委屈感 | 努力不被看见、好心被当驴肝肺 |
| 崩溃 | 🤯 | 情绪到达极限的崩溃感 | 项目上线前夜、连续通宵后 |
| 麻木 | 😴 | 对工作失去感觉的麻木状态 | 重复性劳动久了、职业倦怠期 |
| 想润 | 🔥 | 想逃离当前环境（辞职/转行） | 忍无可忍、看到更好的机会 |
| 焦虑 | 😰 | 对未来的焦虑和不安 | 35岁危机、行业裁员潮 |
| 加油 | 💪 | 积极、互相鼓励的状态 | 拿到offer、项目完成、互相打气 |

### 7.3 牛马等级体系

| 等级 | 名称 | 所需经验值 | 视觉标识 | 达成条件 |
|------|------|------------|----------|----------|
| Lv1 | 小牛马 | 0 | 🐴 默认头像 | 注册即获得 |
| Lv2 | 初级牛马 | 100 | 🐴 铜色边框 | 累计发帖 10+ 或被共情 5+ |
| Lv3 | 中级牛马 | 500 | 🐴 银色边框 | 累计发帖 50+ 且被共情 30+ |
| Lv4 | 老牛马 | 2000 | 🐴 金色边框 | 累计发帖 200+ 且被共情 100+ |
| Lv5 | 牛马之王 | 8000 | 🐴 闪烁金边 + 专属徽章 | 累计发帖 500+ 且被共情 500+ |

> **经验值计算公式（MVP 简化版）：** `经验值 = 发帖数 × 2 + 被共情数 × 5 + 登录天数 × 1`
