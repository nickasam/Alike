## 1. 注册登录

### US-01：新用户注册

**As a** 牛马，
**I want** 用邮箱和密码注册账号，
**so that** 我能加入 Alike 社区，找到懂我辛苦的人。

**验收条件：**
1. 注册页面提供邮箱、密码、确认密码、昵称四个必填字段
2. 邮箱格式校验不通过时，提示"请输入有效的邮箱地址"
3. 密码长度不少于 8 位，须包含字母和数字
4. 两次密码不一致时，提示"两次输入的密码不一致"
5. 邮箱已被注册时，提示"该邮箱已注册，请直接登录"
6. 昵称长度 2-20 字符，支持中英文
7. 注册成功后，自动登录并跳转到首页
8. 密码以 bcrypt 加密存储，数据库中不存明文

---

### US-02：用户登录

**As a** 牛马，
**I want** 用邮箱和密码登录，
**so that** 我能进入自己的社区账号，继续和牛马兄弟姐妹们交流。

**验收条件：**
1. 登录页面提供邮箱、密码两个字段
2. 输入正确凭证后，返回 JWT token 并跳转到首页
3. 密码错误时，提示"邮箱或密码错误"（不泄露哪个字段错）
4. 账号不存在时，同样提示"邮箱或密码错误"
5. 登录后刷新页面，登录状态保持（JWT 存储于 localStorage/cookie）
6. JWT 过期后，自动调用 refresh 接口续期；续期失败则跳转登录页
7. 连续 5 次密码错误后，该 IP 触发限流（15 分钟内禁止再试）

---

### US-03：退出登录

**As a** 牛马，
**I want** 随时退出登录，
**so that** 在公用电脑上使用时保护我的账号安全。

**验收条件：**
1. 顶部导航栏显示"退出"按钮
2. 点击退出后，清除本地 JWT token，跳转到登录页
3. 退出后，WebSocket 连接自动断开
4. 退出后再访问需要认证的页面，自动跳转登录页

## 2. 发帖 / 回复 / 线程

### US-04：在频道中发消息

**As a** 牛马，
**I want** 在频道中发一条消息，
**so that** 我能吐槽今天的工作、分享经验或提问求助。

**验收条件：**
1. 频道页底部显示消息输入框，支持多行文本
2. 输入框支持 Enter 发送、Shift+Enter 换行
3. 消息内容不能为空，空消息发送时提示"说点什么吧"
4. 单条消息不超过 2000 字符，超出时提示长度限制
5. 发送后消息立即显示在消息流中（乐观更新）
6. 消息通过 WebSocket 实时推送到该频道所有在线成员
7. 支持上传图片和文档附件（图片 ≤5MB，文档 ≤10MB，图片限 jpg/png/gif/webp）
8. 消息发送失败时（网络错误），显示重试按钮

---

### US-05：线程回复

**As a** 牛马，
**I want** 对某条消息发起或参与线程讨论，
**so that** 跑题的深入讨论不会打断频道主消息流。

**验收条件：**
1. 每条消息右侧显示"回复"按钮，点击后展开线程面板
2. 线程面板显示原消息 + 所有回复，按时间正序排列
3. 在线程面板中输入回复并发送，回复实时显示
4. 线程回复通过 WebSocket 推送 `thread_reply` 事件
5. 原消息上显示回复计数，计数 > 0 时可点击展开
6. 线程面板可关闭，关闭后不影响频道消息流
7. 回复同样支持匿名选项和情绪标签

---

### US-06：删除自己的消息

**As a** 牛马，
**I want** 删除自己发布的消息，
**so that** 我可以撤回说错的话或发错的内容。

**验收条件：**
1. 只有消息发布者可以删除自己的消息
2. 消息上显示"删除"按钮（仅本人可见）
3. 点击删除后弹出确认对话框，二次确认
4. 删除后消息从消息流中移除，显示"该消息已被删除"占位
5. 线程回复同样支持删除
6. 删除操作通过 WebSocket 同步到所有在线用户
7. 频道管理员可以删除任何人的违规消息

## 3. 加入频道

### US-07：浏览和加入频道

**As a** 牛马，
**I want** 浏览频道列表并加入感兴趣的频道，
**so that** 我能找到和自己同行业/岗位的牛马们，快速融入圈子。

**验收条件：**
1. 侧边栏显示频道列表，按分类（行业/岗位/话题/自定义）分组
2. 每个频道显示名称、图标、成员数、最近消息摘要
3. 点击频道可预览频道详情（描述、成员数、近期活跃度）
4. 频道详情页有"加入频道"按钮
5. 加入后，频道出现在"我的频道"列表中
6. 加入频道后，频道消息实时推送（WebSocket 订阅）
7. 支持搜索频道名称快速定位

---

### US-08：离开频道

**As a** 牛马，
**I want** 离开不再感兴趣的频道，
**so that** 我的频道列表保持整洁，减少无关消息打扰。

**验收条件：**
1. 已加入的频道在频道详情页显示"离开频道"按钮
2. 点击离开后弹出确认提示
3. 离开后，该频道移出"我的频道"列表
4. 离开后不再接收该频道的 WebSocket 消息推送
5. 离开后历史消息仍可通过重新加入查看
6. 频道管理员离开时，若该频道无其他管理员，系统提示先转移管理员权限

## 4. 情绪标签

### US-09：发消息时选择情绪标签

**As a** 牛马，
**I want** 发消息时给消息打一个情绪标签，
**so that** 别人能一眼看出我的心情状态，更容易产生共鸣。

**验收条件：**
1. 消息输入框旁边显示情绪选择器（8 种：😮‍💨疲惫、😡愤怒、😢委屈、🤯崩溃、😴麻木、🔥想润、😰焦虑、💪加油）
2. 情绪标签为可选，不选时消息无标签
3. 每条消息最多选择 1 个情绪标签
4. 发送后，消息上显示情绪标签（图标 + 文字）
5. 情绪标签随消息通过 WebSocket 推送
6. 情绪标签影响消息在情绪看板上的统计

---

### US-10：查看频道情绪看板

**As a** 牛马，
**I want** 查看当前频道的情绪看板，
**so that** 我能感知大家的整体情绪状态，看看今天牛马们过得怎么样。

**验收条件：**
1. 频道页侧边或顶部显示情绪看板组件
2. 看板以柱状图/热力图形式展示当前频道各情绪标签的消息数量分布
3. 数据实时更新（通过 WebSocket `emotion_update` 事件）
4. 点击某个情绪标签，可筛选显示该情绪的消息
5. 看板默认显示当日数据，可切换查看 7 天趋势
6. 情绪数据每 30 秒至少刷新一次

## 5. 共情（抱团取暖）

### US-11：对消息点击共情

**As a** 牛马，
**I want** 对触动我的消息点击"抱团取暖"，
**so that** 让对方知道有人懂他的辛苦，你不是一个人。

**验收条件：**
1. 每条消息下方显示"抱团取暖"按钮（火焰/拥抱图标）
2. 点击后按钮播放暖色动画效果，共情数 +1
3. 每个用户对同一条消息只能共情一次，重复点击为取消
4. 共情数实时更新（通过 WebSocket `empathy` 事件推送）
5. 共情后，消息发布者收到共情通知
6. 已共情状态有视觉区分（按钮高亮/变色）
7. 匿名消息同样可以被共情，但通知不暴露共情者真实身份

---

### US-12：查看共情排行榜

**As a** 牛马，
**I want** 查看共情排行榜，
**so that** 我能看到社区里最温暖的人和最触动人心的故事。

**验收条件：**
1. 排行榜页面展示"最受共情榜"（按用户累计被共情数排序）
2. 排行榜同时展示"最暖牛马榜"（按用户给出的共情数排序）
3. 每条记录显示排名、用户昵称/头像、共情数
4. 支持按时间范围筛选（今日 / 本周 / 本月 / 全部）
5. TOP 10 有特殊样式（金银铜牌图标）
6. 点击用户名可跳转到该用户个人主页
7. 排行榜数据每小时更新一次

## 6. 打工日记 / 打卡

### US-13：写打工日记

**As a** 牛马，
**I want** 写一篇打工日记记录今天的工作经历和心情，
**so that** 我能留下每天的打工痕迹，日后回看也能看到自己的成长或坚持。

**验收条件：**
1. 日记广场页有"写日记"入口
2. 日记编辑器支持标题（必填，≤50字符）和正文（必填，支持富文本/Markdown）
3. 必须选择当天心情（情绪标签）
4. 可选择公开或私密：公开日记出现在日记广场，私密仅自己可见
5. 支持上传封面图片
6. 保存成功后跳转到日记详情页
7. 支持草稿自动保存（每 30 秒）

---

### US-14：打工打卡

**As a** 牛马，
**I want** 每天写日记时自动打卡，
**so that** 我能维持连续打卡的记录，给自己一点小小的仪式感。

**验收条件：**
1. 每天首次发布日记时自动完成当日打卡
2. 用户主页显示连续打卡天数和总打卡天数
3. 打卡日历可视化展示（日历视图，已打卡日期高亮）
4. 连续打卡达到里程碑（7天/30天/100天/365天）时，弹出庆祝提示
5. 打卡记录不可补签（当天 24:00 前未打卡则断签）
6. 断签后连续天数归零，总天数不变
7. 打卡数据通过 WebSocket 通知用户本人

---

### US-15：浏览日记广场

**As a** 牛马，
**I want** 在日记广场浏览其他牛马的公开日记，
**so that** 我能看看大家都在经历什么，找到共鸣和慰藉。

**验收条件：**
1. 日记广场以信息流形式展示公开日记
2. 每条日记卡片显示标题、作者、心情标签、发布时间、共情数、评论数
3. 支持按心情标签筛选日记
4. 支持按时间排序（最新 / 最热）
5. 分页加载，每页 20 条，滚动到底部自动加载下一页
6. 点击日记卡片进入日记详情页
7. 日记详情页支持评论和共情

## 7. 通知

### US-16：收到共情和回复通知

**As a** 牛马，
**I want** 在别人共情我的消息或回复我的帖子时收到通知，
**so that** 我能及时知道有人在意我、回应我，不会错过互动。

**验收条件：**
1. 顶部导航栏显示通知铃铛图标，带未读数角标
2. 通知类型包括：共情通知、回复通知、@提及通知、系统通知
3. 点击铃铛展开通知下拉列表，显示最近 20 条通知
4. 每条通知显示类型图标、摘要内容、时间
5. 点击通知跳转到对应消息/日记/频道
6. 新通知通过 WebSocket 实时推送，铃铛动画提示
7. 已读通知和未读通知有视觉区分
8. 支持一键标记全部已读

---

### US-17：通知设置管理

**As a** 牛马，
**I want** 自定义接收哪些类型的通知，
**so that** 我不会被过多通知打扰，只收我在意的。

**验收条件：**
1. 设置页提供通知偏好开关：共情通知、回复通知、@提及通知、系统通知
2. 每种通知类型可独立开关
3. 设置变更即时生效，无需额外保存按钮
4. 关闭某类通知后，该类型不再产生站内通知和 WebSocket 推送
5. 系统通知不可关闭（如账号安全、版本公告）
6. 设置存储在服务端，跨设备同步

## 8. 搜索

### US-18：全文搜索消息

**As a** 牛马，
**I want** 搜索频道内的消息关键词，
**so that** 我能快速找到之前看过的讨论、经验分享或求助帖。

**验收条件：**
1. 顶部导航栏提供全局搜索框
2. 搜索支持关键词匹配消息内容
3. 搜索结果按相关度排序，显示消息摘要、所属频道、作者、时间
4. 支持按频道、时间范围、情绪标签筛选结果
5. 搜索结果分页，每页 20 条
6. 仅搜索用户已加入频道的消息（权限隔离）
7. 搜索关键词高亮显示
8. 无结果时显示"没有找到相关内容，换个关键词试试？"

---

### US-19：搜索频道和用户

**As a** 牛马，
**I want** 搜索频道名称或用户昵称，
**so that** 我能快速找到想加入的频道或想关注的人。

**验收条件：**
1. 搜索框支持切换搜索类型：消息 / 频道 / 用户
2. 频道搜索结果显示频道名称、描述、成员数、分类
3. 用户搜索结果显示昵称、头像、行业/岗位、牛马等级
4. 频道结果可直接点击"加入"
5. 用户结果可直接点击跳转个人主页
6. 搜索支持模糊匹配和拼音首字母
7. 搜索响应时间 < 500ms

## 9. 匿名发布

### US-20：匿名发消息

**As a** 牛马，
**I want** 在频道中匿名发消息，
**so that** 我能放心吐槽敏感话题，不用担心暴露身份被公司找人。

**验收条件：**
1. 消息输入框旁有"匿名发送"开关
2. 开启匿名后，消息在消息流中显示"匿名牛马"和默认头像，不显示真实昵称/头像
3. 匿名消息同样支持情绪标签和共情
4. 匿名消息的作者信息在数据库中仍记录真实 user_id（用于审计和删除权限），但 API 返回时不暴露
5. 消息发布者本人查看自己的匿名消息时，显示"（我 · 匿名）"标记
6. 频道管理员无法直接查看匿名消息作者，但可删除违规匿名消息
7. 同一用户在同一频道短时间内的匿名消息使用一致的匿名标识（如"匿名牛马#3F2A"），以便区分不同匿名者

---

### US-21：个人主页匿名设置

**As a** 牛马，
**I want** 在个人设置中开启全局匿名模式，
**so that** 我发布的所有消息默认匿名，不用每次手动勾选。

**验收条件：**
1. 个人设置页提供"默认匿名发布"开关
2. 开启后，发消息时"匿名发送"默认勾选，可手动取消
3. 设置存储在 users 表的 `anonymous` 字段
4. 设置变更后立即生效，影响后续所有消息
5. 已发布消息的匿名状态不受设置变更影响
6. 日记发布不受全局匿名设置影响（日记默认署名）

## 10. 个人主页

### US-22：查看个人主页

**As a** 牛马，
**I want** 查看自己或他人的个人主页，
**so that** 我能了解这个人的打工背景、牛马等级和社区贡献。

**验收条件：**
1. 个人主页显示：昵称、头像、行业、岗位、工龄、牛马等级
2. 显示统计数据：累计共情数（收到/给出）、打卡天数、消息数、加入频道数
3. 显示连续打卡日历视图
4. 显示该用户公开的打工日记列表（最新 5 篇，可查看全部）
5. 显示加入的频道列表
6. 匿名消息不计入主页的消息展示（不暴露匿名内容关联）
7. 查看自己主页时，额外显示"编辑资料"按钮
8. 他人主页不显示私密日记

---

### US-23：编辑个人资料

**As a** 牛马，
**I want** 编辑我的个人资料，
**so that** 我能展示自己的打工身份，让同行业的人更容易找到我。

**验收条件：**
1. 个人主页点击"编辑资料"进入编辑页
2. 可编辑字段：昵称、头像、行业、岗位、工龄、个人简介
3. 头像支持上传图片（≤2MB，限 jpg/png）
4. 行业为下拉选择（互联网/金融/教育/制造业/服务业/医疗/法律/其他）
5. 工龄为数字输入（0-50 年）
6. 个人简介 ≤ 200 字符
7. 保存后主页信息立即更新
